# AgentHub：从零构建 IM 式多 Agent 协作平台

参加 AI 全栈挑战赛，我选择了一个有挑战性的课题：构建一个类似飞书/微信的多 Agent 协作平台。

## 课题背景

多 Agent 协作是提升复杂任务执行效率的关键趋势。AgentHub 的核心交互范式是 IM 聊天：

- **新建对话**：创建会话，选择 Agent
- **多会话并行**：同时开启多个对话窗口
- **群聊协作**：@ 多个 Agent，Orchestrator 自动协调
- **产物内联**：代码 Diff、网页预览卡片直接嵌入聊天流

## 核心设计

### 1. Orchestrator 三层 Prompt

```
Layer 1: 场景分析 → 判断任务类型和复杂度
Layer 2: 角色生成 → 根据任务生成 Agent 团队
Layer 3: 任务拆解 → 拓扑排序，分配子任务
```

每层都是独立的 LLM 调用，结果可观测。

## 关键实现细节

### Claude Code CLI 集成

```typescript
const process = spawn('claude', [
  '-p', prompt,
  '--output-format', 'stream-json'
], { cwd: workDir })

for await (const chunk of readProcess(process)) {
  yield chunk
}
```

使用子进程管理，支持超时和取消。

### 群聊模式

用户发送 `@所有人` 时，Orchestrator 启动多轮讨论：

1. 第 1 轮：每个 Agent 发表意见
2. 第 2-3 轮：Agent 基于他人发言补充
3. 汇总结果

### 产物内联

代码 Diff 使用 Monaco Editor，网页预览使用 iframe：

```tsx
<iframe
  srcDoc={srcdoc}
  sandbox="allow-scripts allow-same-origin"
/>
```

## 经验总结

1. **Prompt 设计是核心**：三层分离，每层职责清晰
2. **流式输出体验好**：用户实时看到进度，不会焦虑
3. **适配器抽象是关键**：后续接入新平台只需实现接口
4. **IM 范式适合协作**：用户熟悉的交互方式
