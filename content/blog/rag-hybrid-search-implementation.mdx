# RAG 混合搜索实战：关键词 + 向量 + RRF 融合

在个人网站的 Nova AI 助手中，我实现了一套混合搜索系统。没有使用 Elasticsearch 或 Pinecone，纯 TypeScript 实现，效果出奇地好。

## 为什么需要混合搜索？

单一检索方式各有缺陷：

| 方式 | 优势 | 缺陷 |
|------|------|------|
| 关键词检索 | 精确匹配、响应快 | 无法理解语义 |
| 向量检索 | 语义理解、泛化强 | 专业术语召回差 |

混合搜索结合两者优点：精确匹配保底，语义检索补充。

## 核心算法：RRF 融合

Reciprocal Rank Fusion 是一种简单高效的融合方法：

```
score(d) = 1/(k + rank_vector) + 1/(k + rank_keyword)
```

`k` 通常取 60，对排名靠前的文档给予更高权重。

```typescript
function rrfFusion(
  vectorResults: Array<{ docId: string; score: number }>,
  keywordResults: Array<{ docId: string; score: number }>,
  k: number = 60
): Array<{ docId: string; score: number }> {
  const docScores: Map<string, number> = new Map();

  // 向量检索贡献
  for (let rank = 0; rank < vectorResults.length; rank++) {
    const { docId } = vectorResults[rank];
    const score = 1.0 / (k + rank + 1);
    docScores.set(docId, (docScores.get(docId) || 0) + score);
  }

  // 关键词检索贡献
  for (let rank = 0; rank < keywordResults.length; rank++) {
    const { docId } = keywordResults[rank];
    const score = 1.0 / (k + rank + 1);
    docScores.set(docId, (docScores.get(docId) || 0) + score);
  }

  return [...docScores.entries()]
    .map(([docId, score]) => ({ docId, score }))
    .sort((a, b) => b.score - a.score);
}
```

## 中文 N-gram 分词

没有 jieba 分词器？N-gram 是个实用的替代方案：

```typescript
function tokenizeChinese(text: string): string[] {
  const chinesePattern = /[一-龥]+/g;
  const sequences = text.match(chinesePattern) || [];

  const terms: string[] = [];
  for (const seq of sequences) {
    // 短词直接用，长词生成 2-4 字片段
    if (seq.length <= 4) {
      terms.push(seq);
    } else {
      for (let n = 2; n <= 4; n++) {
        for (let i = 0; i <= seq.length - n; i++) {
          terms.push(seq.slice(i, i + n));
        }
      }
    }
  }
  return [...new Set(terms)];
}
```

"人工智能技术" → ["人工", "人工智", "人工智能", "工智", "工智能", "智能", "能技", "能技术", "技术"]

## 关键词检索评分

标题匹配权重更高：

```typescript
function keywordSearch(query: string, docs: Document[]) {
  const queryTerms = tokenizeChinese(query);
  const scores = new Map<string, number>();

  for (const doc of docs) {
    let docScore = 0;
    for (const term of queryTerms) {
      // 标题匹配权重 3.0

      docScore += titleMatches * 3.0;

      // 内容匹配权重 0.5，饱和处理防止刷分
      const contentMatches = (doc.content.match(regex) || []).length;
      docScore += (contentMatches / (1 + contentMatches * 0.1)) * 0.5;
    }
    if (docScore > 0) scores.set(doc.id, docScore);
  }

  return [...scores].sort((a, b) => b[1] - a[1]);
}
```

## 向量检索：伪查询向量

没有在线 Embedding 服务？从 Top-K 关键词结果的平均向量构造伪查询：

```typescript
function computePseudoQueryEmbedding(
  keywordResults: Array<{ docId: string }>,
  docIds: string[],
  vectors: number[][],
  dimension: number,
  topK: number = 3
): number[] {
  const queryEmbedding = new Array(dimension).fill(0);
  const topDocs = keywordResults.slice(0, topK);

  for (const { docId } of topDocs) {
    const idx = docIds.indexOf(docId);
    if (idx !== -1) {
      for (let i = 0; i < dimension; i++) {
        queryEmbedding[i] += vectors[idx][i];
      }
    }
  }

  // 平均
  for (let i = 0; i < dimension; i++) {
    queryEmbedding[i] /= topDocs.length;
  }

  return queryEmbedding;
}
```

这是一种近似方法，但在实践中效果不错。

## 安全防护

所有正则表达式必须转义用户输入：

```typescript
function escapeRegExp(string: string): string {
  return string.replace(/[.*+?^${}()|[\]\\]/g, '\\$&');
}
```

防止 ReDoS 攻击。
