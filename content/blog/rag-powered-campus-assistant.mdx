# RAG 技术实战

RAG (Retrieval-Augmented Generation) 是让 LLM 接入私有知识的关键技术。本文分享校园百事通项目的技术选型和实现。

## 技术选型

### 向量数据库

对比了 ChromaDB、Milvus、Pinecone：

| 特性 | ChromaDB | Milvus | Pinecone |
|------|----------|--------|----------|
| 部署难度 | 简单 | 中等 | 托管 |
| 性能 | 中等 | 高 | 高 |
| 成本 | 免费 | 免费 | 付费 |
| 适用场景 | 开发/小规模 | 生产/大规模 | 生产 |

校园场景数据量小，选择 ChromaDB。

### Embedding 模型

测试了几个模型在中文场景的表现：

- text-embedding-ada-002: 英文强，中文一般
- bge-large-zh: 中文最佳，免费
- m3e-base: 平衡选择

最终选择 bge-large-zh，准确率高且免费。

## 检索策略

### 基础检索

```python
results = vectorstore.similarity_search(query, k=5)
```

### 混合检索

```python
# BM25 + 向量检索
bm25_results = bm25.search(query, k=10)
vector_results = vectorstore.search(query, k=10)
final = rerank(bm25_results, vector_results)
```

### 重排序

```python
# 使用 Cross-Encoder 重排序
from sentence_transformers import CrossEncoder
reranker = CrossEncoder('BAAI/bge-reranker-large')
scores = reranker.predict([(query, doc) for doc in docs])
```

## 实战问题

### 问题 1: 分块策略

校园文档主要是政策公告，整块分割效果好：

```python
text_splitter = RecursiveCharacterTextSplitter(
    chunk_size=500,
    chunk_overlap=50,
    separators=["\\n\\n", "\\n", "。", "；"]
)
```

### 问题 2: 多轮对话

用户会追问，需要保持上下文：

```python
# 将历史问答注入检索
enriched_query = f"{previous_qa}\\n当前问题: {query}"
```

## 效果评估

使用 RAGAS 评估框架：

- 忠实度 (Faithfulness): 0.92
- 答案相关性: 0.88
- 上下文精确度: 0.85

## 总结

RAG 不是银弹，需要根据场景调整。关键是：
1. 选对 embedding 模型
2. 设计合理的分块策略
3. 持续评估和优化
