# SSE 流式输出：让 AI 响应"活"起来

在 AgentHub 项目中，我们使用 Server-Sent Events (SSE) 实现实时流式输出。相比 WebSocket，SSE 更简单，更适合单向推送场景。

## SSE vs WebSocket

| 特性 | SSE | WebSocket |
|------|-----|-----------|
| 方向 | 单向（服务器→客户端） | 双向 |
| 协议 | HTTP | WebSocket |
| 重连 | 浏览器自动 | 手动实现 |
| 复杂度 | 简单 | 复杂 |

AI 响应场景：服务器推送，客户端接收。SSE 完美匹配。

## 服务端实现

### Next.js API Route

```typescript
export async function POST(request: Request) {
  const stream = new ReadableStream({
    async start(controller) {
      const encoder = new TextEncoder()

      function sendEvent(data: { agentId: string; type: string; content: string }) {
        controller.enqueue(encoder.encode(`data: ${JSON.stringify(data)}\n\n`))
      }

      try {
        sendEvent({ agentId: 'orchestrator', type: 'status', content: '分析中...' })

        // 执行任务...
        for await (const chunk of agent.send(task)) {
          sendEvent({ agentId: chunk.agentId, type: 'text', content: chunk.content })
        }

        sendEvent({ agentId: 'orchestrator', type: 'done', content: '完成' })
      } finally {
        controller.close()
      }
    }
  })

  return new Response(stream, {
    headers: {
      'Content-Type': 'text/event-stream',
      'Cache-Control': 'no-cache',
      'Connection': 'keep-alive',
    }
  })
}
```

关键点：
- `data: JSON\n\n` 格式（双换行）
- `Content-Type: text/event-stream`
- `Cache-Control: no-cache`

## 客户端实现

### React Hook

```typescript
export function useChat(sessionId: string | null) {
  const [messages, setMessages] = useState<Message[]>([])
  const [streaming, setStreaming] = useState<Record<string, string>>({})

  const send = useCallback(async (content: string) => {
    const res = await fetch(`/api/sessions/${sessionId}/chat`, {
      method: 'POST',
      body: JSON.stringify({ message: content }),
    })

    const reader = res.body?.getReader()
    if (!reader) return

    const decoder = new TextDecoder()
    let buffer = ''

    while (true) {
      const { done, value } = await reader.read()
      if (done) break

      buffer += decoder.decode(value, { stream: true })
      const lines = buffer.split('\n\n')
      buffer = lines.pop() || ''

      for (const line of lines) {
        if (!line.startsWith('data: ')) continue
        const event = JSON.parse(line.slice(6))

        if (event.type === 'done') {
          setMessages(prev => [...prev, { role: 'agent', content: event.content }])
          setStreaming(prev => { const next = { ...prev }; delete next[event.agentId]; return next })
        } else {
          setStreaming(prev => ({
            ...prev,
            [event.agentId]: (prev[event.agentId] || '') + event.content
          }))
        }
      }
    }
  }, [sessionId])

  return { messages, streaming, send }
}
```

## 错误处理

### 中断请求

```typescript
const controller = new AbortController()

fetch(url, { signal: controller.signal })

// 用户点击停止
controller.abort()
```

### 超时处理

```typescript
const timeout = setTimeout(() => controller.abort(), 60000)

fetch(url, { signal: controller.signal })
  .finally(() => clearTimeout(timeout))
```

### 断线重连

SSE 浏览器自动重连，但需要处理 Last-Event-ID：

```typescript
// 服务端
headers: {
  'Last-Event-ID': lastEventId
}

// 客户端：浏览器自动发送 Last-Event-ID header
```

## 多 Agent 并发流式

当多个 Agent 同时输出时，需要区分来源：

```typescript
sendEvent({ agentId: 'agent-1', type: 'text', content: '...' })
sendEvent({ agentId: 'agent-2', type: 'text', content: '...' })
```

客户端按 `agentId` 分组累积：

```typescript
setStreaming(prev => ({
  ...prev,
  [event.agentId]: (prev[event.agentId] || '') + event.content
}))
```

## 性能优化

### 缓冲区管理

```typescript
// 不好的做法：每次 read 都处理
const { value } = await reader.read()
process(value) // 可能不完整

// 好的做法：用缓冲区拼接
let buffer = ''
while (true) {
  const { value } = await reader.read()
  buffer += decoder.decode(value, { stream: true })
  const lines = buffer.split('\n\n')
  buffer = lines.pop() || ''
  for (const line of lines) process(line)
}
```

### 批量更新

React 状态更新频繁时，使用 `useReducer` 或 `flushSync`。

## 总结

SSE 流式输出让 AI 响应更"活"。实现要点：

1. 服务端：`ReadableStream` + `data: JSON\n\n` 格式
2. 客户端：`reader.read()` 循环 + 缓冲区管理
3. 错误处理：AbortController + 超时
4. 多 Agent：按 ID 分组累积

简单、可靠、体验好。推荐用于 AI 响应场景。
